# ADR 001: Adoption of React 19

## Status
Approved

## Context
When bootstrapping the frontend client for ExpenseFlow, we need to decide on the core React runtime version. React 19 is now the recommended stable version, offering improvements in performance, asynchronous transitions, and native form actions.

## Decision
We will use React 19 and React DOM 19 as the framework base for the frontend client, running on Vite with TypeScript.

## Consequences
- Requires using modern libraries compatible with React 19 (e.g. TanStack Query v5, MUI v7 equivalents, React Hook Form).
- Eliminates the need for manual rendering optimizations by utilizing React 19's compiler-ready patterns.
- Allows native support for Server Actions and Async Transitions in future modules.
